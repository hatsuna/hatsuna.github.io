Run Run Rambit

Game Development Studio December 2015

As the last remaining beings escaping a world-destroying life form, Run Run Rambit pits four players in a local cooperative-competitive struggle to survive. Created for Game Development Studio and inspired by the quirky style of Muscle March, players must take turns charging forward to remove the obstacles corresponding to their color. Charge too fast into an obstacle that doesn’t match, and the player will get knocked back further towards the swirling abyss. However, once a player has been swallowed up and removed from the game, their corresponding obstacles will be removed as well. So coordinate (or don’t) with your fellow life-forms to make sure everyone (or no-one) gets out alive!

Some stuff I did while making this really weird game:

  • Prototyping gameplay and input
  • Game systems code, procedural generation, collisions and interactions
  • Obstacle gameplay design, obstacle-player interaction design
  • Catbro & Giraffe (animal) modeling, texturing, rigging, and animations

Rambit was an interesting project because it pushed me outside my box. I came in relatively new to the Maya side of things and was able to pick up some modeling, animation, etc. to round out the efforts of the group. There was something uniquely satisfying to not only creating code to enable players to act within the game world, but also to create life within the game world that the players would get to control through working on the art side of things. Seeing the two intermingle and interact as a result of work that I did was really a great treat for me, and a great thing to experience for the first time.
